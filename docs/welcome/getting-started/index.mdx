# Getting started

## Quickstart

### 1. Install the Datree CLI

Binary releases can be downloaded from the [releases page](https://github.com/datreeio/datree/releases) or can be installed directly

<Tabs>
  <TabItem value='Linux' label='Linux' default>

  </TabItem>
  <TabItem value='MacOS' label='MacOS'>

    curl https://get.datree.io | /bin/bash

  </TabItem>
  <TabItem value='Windows (PowerShell)' label='Windows (PowerShell)'>

    iwr -useb https://get.datree.io/windows_install.ps1 | iex

  </TabItem>
  <TabItem value='Homebrew' label='Homebrew'>

    brew tap datreeio/datree
    brew install datreeio/datree/datree

  </TabItem>
  <TabItem value='Docker' label='Docker'>

    docker pull datree/datree

  </TabItem>
</Tabs>

### 2. Test a Kubernetes (demo) manifest

Run Datree to test it against a pre-compiled demo file - [`k8s-demo.yaml`](https://raw.githubusercontent.com/datreeio/datree/main/internal/fixtures/kube/k8s-demo.yaml):

<Tabs>
  <TabItem value='Linux' label='Linux' default>

    datree test ~/.datree/k8s-demo.yaml

  </TabItem>
  <TabItem value='MacOS' label='MacOS'>

    datree test ~/.datree/k8s-demo.yaml

  </TabItem>
  <TabItem value='Windows (PowerShell)' label='Windows (PowerShell)'>

    datree.exe test $env:USERPROFILE\.datree\k8s-demo.yaml

  </TabItem>
  <TabItem value='Docker' label='Docker'>

    cat ~/.datree/k8s-demo.yaml | docker run -i datree/datree test -

  </TabItem>
</Tabs>

You will get the following output in your CLI:

![gettingstarted_cli_output](/img/welcome/getting-started/output.png)

### 3. Customize your policy

#### Sign up to access your dashboard

Each policy check uses a "default" policy that includes 21 built-in rules. To configure the policy, switch back to your terminal, and sign up by clicking on the link at the bottom of the execution output:&#x20;

![gettingstarted_signup](/img/welcome/getting-started/signup.gif)

**Follow the link in the test summary:**

![gettingstarted_summary](/img/welcome/getting-started/summary.png)

The link from the CLI will lead to a sign-up page. To make it easier for you, so you will not need to create a new user, you can sign up with your private **GitHub** or **Google** account.

<div
  style={{ display: 'flex', justifyContent: 'center', alignItems: 'center' }}>
  <img
    src='/img/welcome/getting-started/signup.png'

</div>

After sign-up, you'll be automatically redirected to your [Centralized policy](/setup/centralized-policy) to set it according to your needs.

#### Enable/disable built-in rules

![gettingstarted_enable_disable](/img/welcome/getting-started/enable.png)

#### [OPTIONAL] Add your custom rules

In addition to the tool's built-in rules, you can also write any rule that you wish and run it against your Kubernetes configurations to check for violations. Read how to set up custom rules at [Understanding custom rules](/custom-rules/custom-rules-overview).
