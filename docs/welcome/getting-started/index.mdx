# Getting started

Prevent Kubernetes misconfigurations from reaching production (again üò§ )! From code to production, Datree provides an E2E policy enforcement solution to run automatic checks for rule violations.
Datree can be used on the command line, admission webhook, or even as a kubectl plugin to run policies against Kubernetes objects.

:::tip 
**No prerequisites to get started!**
:::

## Quickstart

### 1. Install the Datree CLI

Binary releases can be downloaded from the [releases page](https://github.com/datreeio/datree/releases) or can be installed directly

  <TabItem value='Linux' label='Linux' default>

</Tabs>

### 2. Run a policy check against a Kubernetes manifest

Run the CLI against any Kubernetes YAML file.  
You can use our [demo file](https://raw.githubusercontent.com/datreeio/datree/main/internal/fixtures/kube/k8s-demo.yaml) if you wish:

<Tabs>
  <TabItem value='Linux' label='Linux' default>

    datree test ~/.datree/k8s-demo.yaml

  </TabItem>
  <TabItem value='MacOS' label='MacOS'>

    datree test ~/.datree/k8s-demo.yaml

  </TabItem>
  <TabItem value='Windows (PowerShell)' label='Windows (PowerShell)'>

    datree.exe test $env:USERPROFILE\.datree\k8s-demo.yaml

  </TabItem>
  <TabItem value='Docker' label='Docker'>

    cat ~/.datree/k8s-demo.yaml | docker run -i datree/datree test -

  </TabItem>
</Tabs>

You will get the following output in your CLI:

![gettingstarted_cli_output](/img/welcome/getting-started/output.png){ width=70% }

### 3. Customize your policy

#### Connect your CLI to your dashboard

Follow the link in the output summary:

![gettingstarted_summary](/img/welcome/getting-started/summary.png
 
To start using your dashboard, easily sign-up using your private **GitHub** or **Google** account, no need to enter any personal information.

![gettingstarted_signup](/img/welcome/getting-started/signup.gif){ width=70% }

#### Enable/disable built-in rules

After signing up, you'll be automatically redirected to your [Centralized policy](/setup/centralized-policy) where you can configure it according to your needs:

![gettingstarted_enable_disable](/img/welcome/getting-started/enabledisable.png){ width=70% }

#### [OPTIONAL] Add your custom rules

In addition to the tool's built-in rules, you can also write any rule that you wish and run it against your Kubernetes configurations to check for violations. Read how to set up custom rules at [Understanding custom rules](/custom-rules/custom-rules-overview).
