The namespace _default_ is a saved namespace value in which Kubernetes is deploying all objects without an explicit namespace. Using explicit namespaces instead of the default value makes for clearer boundaries between sets of pods in a cluster. For example, namespaces that represent teams present a clear organization of cluster resources and make configuration overlaps less likely.

**Targeted objects by this rule (types of **`kind`**): **Deployment / Pod / DaemonSet / StatefulSet / ReplicaSet / CronJob / Job

**Complexity:** **medium** (<Complexity/>)

**Policy as code identifier:** WORKLOAD_INCORRECT_NAMESPACE_VALUE_DEFAULT

---

## This rule will fail

If a workload has _default_ set as the namespace name

```yaml
metadata:
  namespace: default
```
