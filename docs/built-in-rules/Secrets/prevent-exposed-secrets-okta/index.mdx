**Policy as code identifier:** ALL_EXPOSED_SECRET_OKTA

---

## This rule will fail

If an Okta **access token** is present anywhere in the config:

```yaml
ACCESS_TOKEN: oktarqg:='"sup=nlgfr=d4iaqpsghn-hzy1od=39f1s60b0j-_qg
```

### Rule output in the CLI

```terminal
$ datree test *.yaml

>> File: failExample.yaml
❌ Prevent exposed Okta secrets in objects [1 occurrence]
💡 Secret data found in config - keep your sensitive data elsewhere to prevent it from being stolen 
```
